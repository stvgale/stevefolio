Details

IQVIA has increasing needs for product design support, and strong request of product development efficiency and experience consistency. However, the existing problem is design solutions were tailored for individual product, UX and UI design displays inconsistency from product to product.

  • UX Designer

  • Tools

  • Zeplin
  • Jira

  • Created and presented the UX design to key stakeholders, prototyping, team collaboration, specs and component design

Challenge

Create a Design System that provides a cohesive user experience, accelerate the design process and help to bring future products to life faster.

PROCESS

  1. 1. Research

  2. 2. Design

  3. 3. Test

Research

In order to better understand the challenge and the bigger system, we studied the individual product needs and business requirements. In order to asses the user's needs, we did A/B testing and interviewing to help better understand how each component relates to the various product user flows.

RESEARCH INSIGHTS

  • 1.

    Users need to feel confident their data will not be corrupted, compromised or lost.

  • 3.

    The design system needs to present information and controls that pertain to each user's scenario.

  • 2.

    Users need to be informed so they always know where they are and how to get where they want to go.

  • 4.

    The design system will need to be scalable.

Design

Each component is wireframed, prototyped and heavily vetted for stakeholder approval. I work directly with UI designers and developers to see the final solutions through.

Test

Once final React components are delivered they are audited against approved designs. Components are then put into a design prototype and tested with real users for quality assurance and ideas for enhancements.

Delivery

I worked closely with UI designers and developers to deliver final components.

RESULTS

  • 1.

    Apollo Design System was completed with around 60 components.

  • 2.

    Each component has usage guidelines and detailed wireframes to clarify the interactions.

  • 3.

    React library completed with all the design system contents.

  • 4.

    Streamlined the product design process.

Learnings/takeaways

  • Design systems need to be scalable.
  • Building a design system is difficult.
  • Building a design system requires collaboration and participation.
